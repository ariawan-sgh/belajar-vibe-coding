# AI Planning Specification

Version: 1.0

Purpose:

Mengatur proses pembuatan dokumen secara berantai
agar seluruh artefak proyek tetap konsisten.

---

# 1. General Rules

Setiap dokumen HARUS:

- menggunakan markdown
- memiliki unique ID
- memiliki version
- memiliki source document
- memiliki traceability

Format:

Document Name:
Version:
Source Documents:
Generated By:
Date:

---

# 2. Single Source of Truth

Business Requirement

Source:
PRD

Functional Requirement

Source:
SRS

Data Structure

Source:
Domain Model
ERD

UI Behavior

Source:
UI Specification

API Behavior

Source:
API Contract

Implementation Detail

Source:
AI Coding Specification

AI tidak boleh membuat requirement baru
yang tidak berasal dari source document.

---

# 3. Traceability Rule

Setiap artefak harus dapat ditelusuri.

Example:

PRD Requirement

REQ-001
User login

↓

SRS

FR-001
User dapat login

↓

API

AUTH-001
POST /auth/login

↓

Sprint

SP-01

↓

Task

TASK-001

↓

Code

auth-service.go

# 5. Forbidden Actions

AI MUST NOT

- invent feature
- invent entity
- invent endpoint
- invent module

Without source document.
