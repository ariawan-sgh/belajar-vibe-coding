# 1. Introduction

## 1.1 Purpose

Dokumen ini berisi spesifikasi kebutuhan perangkat lunak untuk WebGIS Analisis Sebaran Toko dan Pesaing. SRS ini mendefinisikan kebutuhan fungsional dan non-fungsional, struktur data, dan batasan sistem berdasarkan PRD dan Domain Model yang telah ditetapkan.

Sistem ini merupakan platform berbasis web untuk analisis persebaran outlet dan kompetitor menggunakan peta interaktif dan analisis spasial. Sistem mencakup manajemen outlet, manajemen kompetitor, visualisasi GIS, analisis radius, heatmap, coverage analysis, dashboard analytics, dan reporting.

# 2. System Overview

## 2.1 System Description

WebGIS Analisis Sebaran Toko dan Pesaing adalah platform berbasis web yang membantu perusahaan retail, FMCG, distributor, franchise, dan UMKM menganalisis persebaran outlet milik sendiri maupun kompetitor menggunakan peta interaktif dan analisis spasial. Sistem memungkinkan pengguna mengidentifikasi area yang sudah jenuh, area potensial untuk ekspansi, overlap antar toko, market coverage, serta peluang pembukaan lokasi baru berdasarkan data geografis dan demografis.

## 2.2 User Personas

### 2.2.1 Area Manager

Tugas: Mengelola outlet wilayah.  
Kebutuhan: Melihat distribusi outlet, analisis performa wilayah.

### 2.2.2 Business Development

Tugas: Membuka outlet baru.  
Kebutuhan: Analisis lokasi potensial, identifikasi kompetitor.

### 2.2.3 Direksi

Tugas: Pengambilan keputusan strategis.  
Kebutuhan: Dashboard nasional, laporan ekspansi.

### 2.2.4 Franchise Owner

Kebutuhan: Menentukan lokasi franchise baru, analisis pasar lokal.

# 3. Functional Requirements

## 3.1 FR-001: Authentication

### 3.1.1 Description
Pengguna harus login untuk mengakses sistem.

### 3.1.2 Inputs
- Username/Email
- Password

### 3.1.3 Processing
- Validasi kredensial
- Generate JWT token
- Reset password melalui email

### 3.1.4 Outputs
- Token akses
- Status autentikasi

### 3.1.5 Acceptance Criteria
- Login berhasil dengan kredensial valid
- Logout berhasil dan token dibatalkan
- Reset password tersedia

### 3.1.6 Traceability
- Source: PRD FR-001
- Domain Entity: User, Role
- API: AUTH-001 (POST /auth/login), AUTH-002 (POST /auth/logout)

## 3.2 FR-002: User Management

### 3.2.1 Description
Admin dapat mengelola pengguna sistem.

### 3.2.2 Inputs
- Data user (nama, email, username, role)
- UserID untuk update/delete

### 3.2.3 Processing
- Validasi data user
- Assign role (Admin, Manager, Analyst)
- Enkripsi password

### 3.2.4 Outputs
- Data user ter-update
- Konfirmasi operasi

### 3.2.5 Acceptance Criteria
- Admin dapat membuat user baru
- Admin dapat mengubah data user
- Admin dapat menghapus user
- Admin dapat mengubah role user

### 3.2.6 Traceability
- Source: PRD FR-002
- Domain Entity: User, Role
- API: USER-001 (POST /users), USER-002 (PUT /users/:id), USER-003 (DELETE /users/:id)

## 3.3 FR-003: Outlet Management

### 3.3.1 Description
Mengelola data outlet perusahaan termasuk import data massal.

### 3.3.2 Inputs
- Nama Outlet
- Alamat
- Latitude
- Longitude
- Tipe Outlet
- Status (Aktif/Tutup/Renovasi)
- File CSV untuk import

### 3.3.3 Processing
- Validasi koordinat
- Validasi format CSV
- Simpan data outlet
- Bulk insert untuk import

### 3.3.4 Outputs
- Data outlet tersimpan
- Laporan import (berhasil/gagal)

### 3.3.5 Acceptance Criteria
- CRUD outlet lengkap
- Import CSV dengan validasi
- Bulk upload mendukung file besar

### 3.3.6 Traceability
- Source: PRD FR-003
- Domain Entity: Outlet, ImportJob
- API: OUTLET-001 (POST /outlets), OUTLET-002 (GET /outlets), OUTLET-003 (PUT /outlets/:id), OUTLET-004 (DELETE /outlets/:id), OUTLET-005 (POST /outlets/import)

### 3.5.6 Traceability

## 3.6 FR-006: Radius Analysis

### 3.6.1 Description
Menghitung area jangkauan outlet dalam radius tertentu.

### 3.6.2 Inputs
- Outlet ID atau koordinat pusat
- Radius (500m, 1km, 3km, custom)
- Layer data

### 3.6.3 Processing
- Hitung buffer area berdasarkan radius
- Identifikasi overlap area
- Hitung jumlah outlet/kompetitor dalam radius

### 3.6.4 Outputs
- Visualisasi radius pada peta
- Daftar outlet dalam radius
- Area overlap

### 3.6.5 Acceptance Criteria
- Radius dapat diubah (500m, 1km, 3km, custom)
- Overlap area terlihat
- Daftar outlet/kompetitor dalam radius ditampilkan

### 3.6.6 Traceability
- Source: PRD FR-006
- Domain Entity: AnalysisSession, AnalysisResult
- API: ANALYSIS-001 (POST /analysis/radius), ANALYSIS-002 (GET /analysis/radius/:id)

## 3.7 FR-007: Coverage Analysis

### 3.7.1 Description
Mengukur wilayah yang terlayani oleh outlet.

### 3.7.2 Inputs
- Data outlet

### 3.7.3 Processing

### 3.7.4 Outputs

- Visualisasi area pada peta

### 3.7.5 Acceptance Criteria
- Persentase coverage dihitung
- Luas coverage ditampilkan
- Area terlayani dan tidak terlayani terklasifikasi

### 3.7.6 Traceability
- Source: PRD FR-007
- Domain Entity: AnalysisSession, AnalysisResult, CoverageArea
- API: ANALYSIS-003 (POST /analysis/coverage), ANALYSIS-004 (GET /analysis/coverage/:id)

### 3.8.1 Description
Membandingkan posisi outlet dengan kompetitor.

### 3.8.2 Inputs
