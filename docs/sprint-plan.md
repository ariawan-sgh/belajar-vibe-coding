## 1. Tujuan Dokumen

Sprint Plan ini membagi MVP **WebGIS Analisis Sebaran Toko dan Pesaing** menjadi sprint yang dapat dieksekusi berdasarkan:

- `docs/product-backlog.md` untuk user story, prioritas, estimasi story point, dan urutan sprint suggestion.
- `docs/vertical-slice-plan.md` untuk vertical slice, dependency, scope API/UI/service/repository/database, seed data, dan Definition of Done.
- `docs/ai-planning-spec.md` untuk aturan dokumen, unique ID, version, source document, dan traceability.

Dokumen ini tidak menambahkan fitur, entitas, endpoint, modul, atau requirement baru di luar dokumen sumber.

## 2. Prinsip Sprint

1. Sprint dijalankan mengikuti dependency sequence dari `docs/vertical-slice-plan.md`.
2. Setiap sprint harus menghasilkan vertical slice yang dapat didemokan secara mandiri.
3. Setiap sprint wajib mencakup UI, API, service, repository, database, seed data, dan skenario demo sesuai slice terkait.
4. Semua fitur authenticated dan authorized wajib menggunakan JWT dan RBAC.
5. Tidak ada endpoint, entity, field, modul, atau fitur baru di luar source document.
6. Story point yang digunakan pada Sprint Plan ini dihitung dari estimasi story point masing-masing user story di Product Backlog.

Catatan: Product Backlog menyebutkan total story point pada Sprint Suggestion, tetapi penjumlahan aktual story point per story tidak sama dengan total yang tertulis. Plan ini menggunakan penjumlahan aktual dari story point per story.

## 11. Definition of Done Global

Setiap sprint dianggap selesai jika:

1. UI dapat dibuka dan digunakan sesuai RBAC.
2. API dapat dipanggil dan mengembalikan response sesuai contract.
3. Service menjalankan validasi dan business logic.
4. Repository mengakses database sesuai ERD.
5. Database memiliki table, index, dan seed data demo.
6. Skenario demo dapat dijalankan tanpa menunggu sprint berikutnya.
7. Tidak ada endpoint, entity, field, modul, atau fitur baru di luar dokumen sumber.

## 12. Batasan dan Catatan

1. `POST /competitors/import` tidak ditambahkan karena tidak ada dalam API Contract saat ini.
2. Import kompetitor tetap tercatat sebagai kebutuhan UI/SRS, tetapi implementasi API perlu disesuaikan terlebih dahulu dengan dokumen sumber sebelum endpoint baru dibuat.
3. Semua koordinat menggunakan WGS84 / EPSG:4326.
4. Query spasial menggunakan PostgreSQL/PostGIS dan spatial index pada `outlets.geom` serta `competitors.geom`.
5. Password user demo tidak boleh hardcoded di kode; gunakan environment variable atau migration seed aman.
