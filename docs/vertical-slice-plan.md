## 1. Tujuan Dokumen

Dokumen ini membagi MVP **WebGIS Analisis Sebaran Toko dan Pesaing** menjadi vertical slice yang dapat di-demo secara independen.

Setiap slice mencakup:

- UI
- API
- Service
- Repository
- Database

Prinsip utama:

1. Tidak menambahkan fitur, entitas, endpoint, atau modul baru di luar dokumen sumber.
2. Setiap slice memiliki alur demo mandiri dengan seed data minimal.
3. Slice mengikuti dependency dari `docs/feature-dependency.md`.
4. Slice setelah authentication tetap menggunakan JWT dan RBAC sesuai source document.
5. Jika sebuah slice membutuhkan data dari slice sebelumnya, slice tersebut menyediakan seed data demo atau skenario data awal.

## 3. Aturan Demo Independen

Setiap slice harus dapat didemokan tanpa menunggu slice berikutnya selesai.

### 3.1 Kebutuhan Umum Demo

Setiap slice memiliki:

- UI yang dapat dibuka di browser.
- API endpoint yang dapat dipanggil.
- Service logic yang dapat dijalankan.
- Repository method yang dapat mengakses database.
- Database table dan seed data minimal.
- Skenario demo yang jelas.

### 3.2 Seed Data Minimal

Seed data wajib disediakan per slice jika data tersebut dibutuhkan untuk demo:

- Role: `Admin`, `Manager`, `Analyst`.
- User demo: minimal 1 admin aktif.
- Outlet demo: minimal 3 outlet dengan koordinat valid.
- Competitor demo: minimal 3 kompetitor dengan koordinat valid.
- Layer config demo: konfigurasi default user.
- Analysis session/result demo: minimal 1 hasil analisis selesai untuk reporting.

Password user demo tidak boleh hardcoded di kode. Gunakan environment variable atau migration seed aman.

## 14. Catatan Implementasi

1. `POST /competitors/import` tidak ditambahkan karena tidak ada dalam API Contract saat ini.
2. Import kompetitor tetap tercatat sebagai kebutuhan UI/SRS, tetapi implementasi API perlu disesuaikan terlebih dahulu dengan dokumen sumber sebelum endpoint baru dibuat.
3. File storage untuk import dan report mengikuti `file_path` pada `import_jobs` dan `reports`.
4. Semua koordinat menggunakan WGS84 / EPSG:4326.
5. Query spasial menggunakan PostgreSQL/PostGIS dan spatial index pada `outlets.geom` serta `competitors.geom`.
