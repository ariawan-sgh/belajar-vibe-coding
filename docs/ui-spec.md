## 2.2 UI-001: Login Screen

### Purpose

Memungkinkan pengguna masuk ke sistem untuk memperoleh akses sesuai role.

### Traceability

- Source: SRS FR-001
- Entities: User, Role
- API: `POST /auth/login`

### Layout

- Logo / nama aplikasi
- Form login
- Link reset password
- Area pesan error

### Fields

| Field | Type | Required | Validation |
|-------|------|----------|------------|
| Username / Email | Text | Yes | Tidak boleh kosong |
| Password | Password | Yes | Tidak boleh kosong |

### Behavior

1. Pengguna memasukkan username/email dan password.
2. Sistem mengirim kredensial ke `POST /auth/login`.
3. Jika berhasil, UI menyimpan status autentikasi dan mengarahkan pengguna ke App Shell.
4. Jika gagal, UI menampilkan pesan error yang jelas.
5. Link reset password mengarah ke proses `POST /auth/reset-password`.

### RBAC

Tidak berlaku karena halaman publik.

### Purpose

### Traceability

- Source: SRS FR-005, FR-006, FR-007, FR-008, FR-009, FR-010
- Entities: LayerConfig, Outlet, Competitor, AnalysisSession, AnalysisResult, CoverageArea
- API: `GET /map/layers`, `PUT /map/layers`, Analysis API, Dashboard API

### Layout

- Panel layer di sisi kiri atau kanan
- Peta utama di area konten
- Panel detail / popup saat marker diklik
- Panel hasil analisis saat analisis dijalankan
- Area pesan status proses analisis

### Map Elements

| Element | Source | Behavior |
|---------|--------|----------|
| Marker Outlet | Outlet | Ditampilkan sesuai layer aktif |
| Marker Kompetitor | Competitor | Ditampilkan sesuai layer aktif |
| Area Administrasi | CoverageArea / area administrasi | Ditampilkan sesuai layer aktif |
| Radius Buffer | AnalysisResult | Ditampilkan setelah Radius Analysis |
| Coverage Area | AnalysisResult, CoverageArea | Ditampilkan setelah Coverage Analysis |

4. Pengguna dapat mengaktifkan atau menonaktifkan layer.
5. Pengguna dapat melakukan zoom dan pan.
6. Klik marker menampilkan popup informasi.
7. Analisis yang selesai ditampilkan sebagai overlay pada peta.
8. Hasil analisis ditampilkan juga dalam panel tabel/ringkasan.

### RBAC

Admin, Manager, Analyst.

## 2.9 UI-008: Analysis Workspace

### Purpose

Menyediakan antarmuka untuk menjalankan dan menampilkan hasil analisis spasial.

### Traceability

- Source: SRS FR-006, FR-007, FR-008, FR-009, FR-010
- Entities: AnalysisSession, AnalysisResult, CoverageArea
- API:
  - `POST /analysis/radius`, `GET /analysis/radius/:id`
  - `POST /analysis/coverage`, `GET /analysis/coverage/:id`
  - `POST /analysis/competitor`, `GET /analysis/competitor/:id`
  - `POST /analysis/heatmap`, `GET /analysis/heatmap/:id`
  - `POST /analysis/whitespot`, `GET /analysis/whitespot/:id`

### Analysis Types

| Analysis Type | UI-008 Section | Required Input | Output |
|---------------|----------------|----------------|--------|
| Radius Analysis | Radius | Outlet ID atau koordinat pusat, radius | Buffer radius, daftar outlet/kompetitor dalam radius, area overlap |
| Coverage Analysis | Coverage | Data outlet, area administrasi, parameter radius | Persentase coverage, luas coverage, area terlayani/tidak terlayani |
| Competitor Analysis | Competitor | Data outlet, data kompetitor, radius analisis | Jarak ke kompetitor terdekat, jumlah kompetitor dalam radius, ranking area |
| Heatmap Analysis | Heatmap | Layer data outlet/kompetitor, parameter intensitas | Visualisasi kepadatan |
| White Spot Analysis | White Spot | Data outlet, area administrasi, parameter minimal outlet | Daftar area potensial, ranking area |

### Behavior

1. Pengguna memilih jenis analisis.
2. UI menampilkan form input sesuai jenis analisis.
3. Saat analisis dijalankan, UI menampilkan status `Proses`.
4. Jika analisis selesai, UI menampilkan overlay hasil pada peta dan tabel ringkasan.
5. Jika analisis gagal, UI menampilkan pesan error.
6. Hasil analisis dapat digunakan untuk generate laporan pada UI-009.

### RBAC

Admin, Manager, Analyst.

## 3.5 Map Interaction Rules

1. Marker outlet dan kompetitor hanya ditampilkan sesuai layer aktif.
2. Popup hanya menampilkan data yang tersedia pada entitas terkait.
3. Overlay analisis hanya ditampilkan setelah status analisis `Selesai`.
4. Jika status analisis `Gagal`, UI menampilkan error state dan tidak menampilkan overlay hasil.
5. Jika data terlalu besar, UI tetap mengikuti target performa peta terbuka < 3 detik sesuai NFR-001.
6. Peta tidak menambahkan fitur di luar analisis yang ada di SRS.

# 6. Acceptance Criteria

## 6.1 UI Acceptance Criteria

- Login dapat dilakukan menggunakan username/email dan password.
- Navigasi ditampilkan sesuai role Admin, Manager, dan Analyst.
- Dashboard menampilkan total outlet, outlet aktif, outlet per wilayah, kompetitor per wilayah, dan coverage percentage.
- Outlet Management mendukung CRUD outlet dan import data outlet.
- Competitor Management mendukung CRUD kompetitor dan import data kompetitor.
- User Management hanya dapat diakses Admin.
- Analysis Workspace menampilkan input dan hasil untuk Radius, Coverage, Competitor, Heatmap, dan White Spot Analysis.
- Reporting mendukung export PDF dan Excel.
- Error message ditampilkan dengan jelas saat validasi atau API gagal.

## 6.2 Map Acceptance Criteria

- Peta interaktif menggunakan OpenLayers.
- Peta menampilkan marker outlet dan kompetitor.
- Popup informasi muncul saat marker diklik.
- Zoom dan pan berfungsi.
- Layer dapat diaktifkan dan dinonaktifkan.
- Overlay analisis ditampilkan setelah hasil analisis selesai.
- Peta memenuhi target performa terbuka dalam < 3 detik.

## 6.3 Layer Acceptance Criteria

- Layer Outlet Sendiri mengikuti konfigurasi `outlet_tampil`.
- Layer Kompetitor mengikuti konfigurasi `kompetitor_tampil`.
- Layer Area Administrasi mengikuti konfigurasi `area_admin_tampil`.
- Layer Kecamatan dan Kelurahan tersedia sebagai layer visual sesuai Domain Model.
- Layer analisis hanya ditampilkan ketika hasil analisis tersedia dan statusnya selesai.
- Konfigurasi layer persisten dikirim melalui `PUT /map/layers`.
