## 1. Product Context

Roadmap implementasi MVP untuk **WebGIS Analisis Sebaran Toko dan Pesaing** berdasarkan PRD, dependency matrix, dan planning specification.

Teknologi target:

- Frontend: Solid JS, TypeScript, OpenLayers, Tailwind 5
- Backend: Golang, Go Fiber
- Database: PostgreSQL, PostGIS

Cakupan MVP mengikuti dokumen sumber: authentication, user management, outlet management, competitor management, GIS map visualization, radius analysis, coverage analysis, competitor analysis, heatmap, potential area detection, dashboard analytics, dan reporting PDF/Excel.

## 3. Roadmap Detail

### 1. Foundation

**Estimasi:** 5-7 hari

**Tujuan:** Menyiapkan landasan teknis agar semua modul dapat dikembangkan dengan standar arsitektur yang konsisten.

**Cakupan pekerjaan:**

- Inisialisasi struktur backend Golang dengan Go Fiber.
- Inisialisasi struktur frontend Solid JS dengan TypeScript dan Tailwind 5.
- Konfigurasi environment development.
- Setup PostgreSQL dan PostGIS.
- Setup migration database.
- Setup logger, configuration loader, dan error handling dasar.
- Setup autentikasi dasar dan middleware JWT.
- Setup role-based access control awal.
- Setup testing foundation untuk backend dan frontend.
- Setup file handling dasar untuk import CSV dan export report.

**Feature dependency yang dipenuhi:**

- MOD-011 File Handling
- MOD-010 Data Access & GIS Storage
- MOD-001 Authentication
- FR-001 Authentication

**Definition of Done:**

- Backend dapat dijalankan.
- Frontend dapat dijalankan.
- Database PostgreSQL/PostGIS tersedia melalui konfigurasi lokal.
- Migration dapat dijalankan.
- JWT authentication dapat divalidasi.
- Role admin, manager, dan analyst tersedia sesuai PRD.

### 2. Domain Layer

**Estimasi:** 4-6 hari

**Tujuan:** Membuat model bisnis inti yang menjadi sumber kebenaran untuk repository, service, API, dan frontend.

**Cakupan pekerjaan:**

- Definisi domain user, role, permission, outlet, competitor, import record, spatial analysis, dashboard metric, dan report.
- Definisi value object untuk koordinat latitude/longitude, radius, coverage metric, dan competitor distance.
- Definisi domain service untuk validasi koordinat, validasi radius, dan perhitungan metrik spasial dasar.
- Definisi boundary antara domain, repository, dan service.
- Validasi data sesuai PRD:
  - Outlet: Outlet ID, nama, alamat, latitude, longitude, tipe outlet, status.
  - Competitor: nama toko, brand, latitude, longitude, kategori.

**Feature dependency yang dipenuhi:**

- FR-002 User Management
- FR-003 Outlet Management
- FR-004 Competitor Management
- FR-006 Radius Analysis
- FR-007 Coverage Analysis
- FR-008 Competitor Analysis
- FR-009 Heatmap Analysis
- FR-010 Potential Area Detection
- FR-011 Dashboard Analytics
- FR-012 Reporting

**Definition of Done:**

- Semua entitas MVP memiliki model domain.
- Validasi koordinat dan radius berada di domain layer.
- Tidak ada logic bisnis utama yang bocor ke API atau frontend.

- MOD-005 GIS Map Visualization
- MOD-006 Spatial Analysis
- MOD-007 Dashboard Analytics
- MOD-008 Reporting
- MOD-009 Import Processing
- MOD-010 Data Access & GIS Storage
- MOD-011 File Handling
